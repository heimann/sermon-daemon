# Sermon Daemon

Local-first server telemetry daemon and CLI.

Sermon Daemon stores host metrics, logs, processes, disks, and operational facts in DuckDB and exposes small CLI primitives for humans and AI agents. Use it standalone over SSH, or connect it to [sermon.fyi](https://sermon.fyi) for hosted retention, enrollment, fleet state, and agent-friendly remote access.

The hosted Sermon control plane is optional. The daemon is useful on its own.

## What it collects

- CPU and memory from `/proc`
- Process snapshots from `/proc`
- Disk usage
- systemd journal logs when available
- Local DuckDB storage with retention
- Optional remote push to a Sermon-compatible ingest endpoint

Remote push includes daemon version, host metrics, disks, top processes by CPU, and a capped batch of recent logs. It intentionally omits process command line arguments.

## Install

Standalone local-only install:

```bash
curl -fsSL https://raw.githubusercontent.com/heimann/sermon-daemon/main/install.sh | bash
```

Hosted Sermon install commands are generated by sermon.fyi and include an ingestion key:

```bash
curl -fsSL https://sermon.fyi/install.sh | bash -s -- \
  --server-name my-server \
  --ingestion-key serm_... \
  --server-url https://sermon.fyi
```

The installer creates a systemd service:

- root install: `/opt/sermon`, `/etc/sermon`, `/var/lib/sermon/metrics.db`, `sermon-agent.service`
- user install: `~/.local/opt/sermon`, `~/.config/sermon`, `~/.local/share/sermon/metrics.db`, user `sermon-agent.service`

To update an installed daemon, rerun the installer with the desired `--version`. If an existing config is present and no `--ingestion-key` is provided, the installer preserves the existing config, replaces the binaries, and restarts the systemd service.

## Query local data

```bash
sermon --db ~/.local/share/sermon/metrics.db status
sermon --db ~/.local/share/sermon/metrics.db processes
sermon --db ~/.local/share/sermon/metrics.db metrics
sermon --db ~/.local/share/sermon/metrics.db logs
sermon --db ~/.local/share/sermon/metrics.db query "select * from metrics order by timestamp desc limit 5"
```

When installed with the default user path, the binary is at:

```bash
~/.local/opt/sermon/bin/sermon
```

## Configuration

The daemon reads JSON config from `~/.config/sermon/config.json` by default. CLI flags override config values.

Standalone example:

```json
{
  "db_path": "~/.local/share/sermon/metrics.db",
  "interval": 10,
  "retention": 604800
}
```

Remote push example:

```json
{
  "db_path": "~/.local/share/sermon/metrics.db",
  "interval": 10,
  "retention": 604800,
  "server_url": "https://sermon.fyi",
  "api_key": "serm_abc123_your_secret_here"
}
```

## Build from source

Requirements:

- Linux glibc
- Zig 0.15.2
- DuckDB 1.2.1, installed by `scripts/bootstrap-duckdb.sh`

```bash
mise install
mise run verify
```

Or manually:

```bash
./scripts/bootstrap-duckdb.sh
./scripts/preflight.sh
zig build
LD_LIBRARY_PATH=lib zig build test
```

Run the daemon locally:

```bash
LD_LIBRARY_PATH=lib ./zig-out/bin/sermon-agent --db ./metrics.db --interval 10
```

Query from another terminal:

```bash
LD_LIBRARY_PATH=lib ./zig-out/bin/sermon --db ./metrics.db status
```
